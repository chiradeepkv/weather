How this PWA works 📖

1
HTML — the structure Every element you see (input box, card, buttons) is an HTML tag. We give each one an id so JavaScript can find it later.
2
CSS — the style All the colors, fonts, and layouts are CSS. We use :root variables so changing one line updates the whole theme.
3
fetch() — calling the API fetch(url) sends a request to a server and gets back data. We use the free Open-Meteo + Geocoding API — no key needed!
4
async / await — waiting for data APIs take time. async / await tells JavaScript to pause and wait for the response before moving on.
5
manifest.json — makes it a PWA This file tells the browser the app's name, icon, and colors. It's what enables the "Add to Home Screen" install prompt.
6
Service Worker — offline power The sw.js file runs in the background. It caches files so the app still opens when there's no internet.
